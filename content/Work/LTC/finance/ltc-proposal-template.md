# LTC Proposal Template

> Generate proposals from this template after needs assessment.
> All proposals are DRAFTS until human review and approval.
> Prices reference [[Work/Shared/pricing-catalog\\|Pricing Catalog]].

---

## Proposal Workflow

1. **Trigger:** Needs assessment completed, offer fit confirmed
2. **Draft:** Populate proposal from this template
3. **Review:** Human reviews and approves pricing, scope, compliance terms
4. **Send:** Human triggers send (may require vendor registration first)
5. **Track:** Move lead to "Proposed" stage in [[LTC Pipeline]]

---

## Proposal Document Structure

### Header
```
Lucido Technology Consulting
[District Name] — [Project Name]
Proposal Date: [YYYY-MM-DD]
Valid Through: [YYYY-MM-DD + 60 days]
RFP Reference: [if applicable]
```

### 1. Executive Summary
- District context (size, current systems, stated needs)
- Problem statement (what's not working)
- Proposed approach (high-level)

### 2. Scope of Work
| Phase | Activities | Deliverables | Duration |
|-------|-----------|-------------|----------|
| Discovery | Stakeholder interviews, system audit, data quality assessment | Audit report + recommendations | 1–2 weeks |
| Design | Architecture, data model, integration plan | Technical design document | 1–3 weeks |
| Implementation | Build, migrate, test | Working system + documentation | 2–8 weeks |
| Training | Staff workshops, handoff | Training materials + sessions | 1 week |
| Support | Post-launch monitoring, adjustments | Support period | 30–90 days |

### 3. Investment
| Item | Price |
|------|-------|
| [Service description] | $[from Pricing Catalog] |
| [Add-on if applicable] | $[from Pricing Catalog] |
| **Total** | **$[sum]** |
| Ongoing Support (optional) | $[2,500–5,000]/month |

### 4. Timeline
- Start: [date, contingent on PO/contract]
- Milestone 1: [date] — [deliverable]
- Milestone 2: [date] — [deliverable]
- Completion: [date]

### 5. Payment Schedule
| Milestone | Amount | Trigger |
|-----------|--------|---------|
| Project Start | 30% | PO issued / contract signed |
| Mid-Project Review | 40% | Deliverable accepted |
| Completion | 30% | Final delivery + training complete |

### 6. Compliance
- FERPA compliance: all student data handled per district data sharing agreement
- AB 1584: contract includes required pupil data provisions
- CALPADS: submissions validated against current CALPADS flash issues
- Insurance: COI provided upon request

### 7. Assumptions & Exclusions
- District provides timely access to systems and stakeholders
- Data exports available in standard formats (CSV, SQL dump, API)
- Scope changes outside this document priced separately at $200/hr

---

## Anti-Patterns
- Do NOT quote before needs assessment
- Do NOT skip compliance language — FERPA/AB 1584 is mandatory
- Do NOT send without human approval
- Do NOT propose without confirming vendor registration requirements
